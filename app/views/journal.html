March 2, 2015: Binary Tree.

I built a binary tree visualization out of NPashaP's graceful tree code So far, it allows the user to add and remove nodes. I intend to add depth first and bredth first searches as well as node coloring. I am concerned that this binary search visualization, while a good demonstration of a binary tree, does not expose the underlying algorithms visually.

April 2, 2015: Coloring for Binary Tree.

Now, when adding or removing a node from the binary tree, one can watch as the path is traced out by a D3 animation. D3 does not have a lot of support for long sequences of transitions in animation, particularly when it's unclear how many steps it will take. In my case, I'm able to calculate timing as I find my node, and use this for visualizing afterwards.

April 7, 2015: Red Black Trees: Every non-leaf node has two children.

In order to make the red-black tree visualization, I've set up a binary tree implementation in which leaves are represented as empty nodes. Every non empty node has 2 children, where one or both of these children may be leaves. It is important that the viewer can see the leaf nodes, because they are colored, and are often players in the rebalancing actions.

April 10, 2015: Red Black Trees: Insertion

Insertion is complete. Debugging was very difficult, but it forced me to understand the actual algorithm, rather than just translate the C implementation on Wikipedia into JavaScript. The out of the box animation, using the D3 from the binary search tree, looks decent, but I will definitely want to make it better, in order to show the actual algorithm better. I'm toying with the idea of displaying some of the code, or pseudocode, on the page, so that viewers can follow along.
